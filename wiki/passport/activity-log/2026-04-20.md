- Codex 주도 작업

### 요약
`webapp/frontend-react`의 단일 번들 경고를 줄이기 위해 route-level code splitting을 적용했다. 각 페이지를 `React.lazy()`로 분리하고 `Suspense` fallback을 추가해 초기 진입 번들을 줄이면서 기존 보호 라우트 동작은 유지했다.

### 작업 내용
- `App.tsx`
  - 페이지 컴포넌트를 eager import에서 `lazy(() => import(...))`로 전환
  - `Suspense` + `Spinner` fallback 추가
  - 보호 라우트를 `ProtectedPage` 래퍼로 정리해 중복된 `RequireAuth + Layout` 구성을 하나로 통일
- 기대 효과
  - 로그인/랜딩 진입 시 전체 업무 페이지를 한 번에 묶어 싣지 않음
  - Vite chunk warning 자체를 낮추고 후속 code splitting 여지를 확보

### 변경 파일(핵심)
- `webapp/frontend-react/src/App.tsx`

### 검증
- `cd webapp/frontend-react && npm run build`
  - 결과: 성공
  - 효과: main JS chunk가 기존 대비 감소하고 route-level split bundle이 생성됨
- `python3 scripts/verify_wiki.py`
  - 결과: PASS

### 미완료
- 가장 큰 상세 페이지(`PassportDetailPage`)와 모달 집합은 여전히 무거워서, 추가 최적화가 필요하면 상세 페이지 내부 lazy 분리까지 검토 가능
- Vite chunk warning이 완전히 사라졌는지는 후속 build 결과에 따라 추가 확인 필요

## Session 3 — 2026-04-20 (Codex)

### 작업 주체
- Codex 주도 작업

### 요약
`webapp/frontend-react`의 후속 code splitting 라운드로 `QrScanPage`와 `PassportDetailPage` 내부 의존성을 분리했다. QR 스캐너 라이브러리는 사용자 액션 시점으로 지연 로드하고, 여권 상세의 탭/모달 묶음은 활성화 시점에만 chunk를 내려받도록 바꿔 초기 상세 진입 비용을 줄였다.

### 작업 내용
- `QrScanPage.tsx`
  - `html5-qrcode` 정적 import 제거
  - 카메라 열기 시점에만 `import('html5-qrcode')`를 수행하도록 변경
  - 버튼 hover/focus에서 scanner module warm-up 추가
- `PassportDetailPage.tsx`
  - `IdentityTab`, `ComplianceTab`, `TraceabilityTab`, `DataTab`, `TrustTab`를 `React.lazy()`로 분리
  - 상세 페이지의 각 업무 모달을 active modal 기준으로만 lazy import하도록 변경
  - 한 번에 모든 탭/모달 코드를 싣지 않도록 `Suspense` 경계 추가
- 기대 효과
  - `QrScanPage` route chunk에서 QR 라이브러리 본체를 제거해 초기 진입 chunk를 대폭 축소
  - `PassportDetailPage` route chunk를 줄이고 탭/모달을 세부 chunk로 분리

### 변경 파일(핵심)
- `webapp/frontend-react/src/pages/QrScanPage.tsx`
- `webapp/frontend-react/src/pages/PassportDetailPage.tsx`
- `wiki/passport/activity-log/2026-04-20.md`

### 검증
- `cd webapp/frontend-react && npm run build`
  - 결과: 성공
  - 확인: `QrScanPage` chunk가 `380.33 kB` → `11.89 kB`로 감소하고 QR 라이브러리는 별도 `esm-*.js` on-demand chunk로 분리됨
  - 확인: `PassportDetailPage` chunk가 `53.49 kB` → `16.76 kB`로 감소하고 탭/모달 chunk가 개별 생성됨
- `bash scripts/clean-runtime-artifacts.sh`
  - 결과: build 산출물 정리 완료
- `python3 scripts/verify_wiki.py`
  - 결과: PASS

### 미완료
- `html5-qrcode` 본체 chunk(`esm-*.js`)는 여전히 크지만 이제 QR 스캔 액션 전까지는 내려받지 않음
- 상세 페이지 내부 데이터 fetch 최적화나 탭 prefetch 전략은 필요 시 다음 라운드에서 별도로 검토 가능

### 교훈
- route-level lazy loading 다음에는 사용자 액션 기반 dynamic import가 체감 성능 개선에 더 직접적이다.
- 상세 페이지는 “한 화면의 모든 도구를 한 번에 import”하지 않고, 현재 탭/현재 모달 기준으로 끊어야 번들 경고를 안정적으로 낮출 수 있다.
