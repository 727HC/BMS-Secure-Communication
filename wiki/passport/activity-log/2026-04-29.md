## Session 3 — AES/CMAC/Ed25519 카드 활성화 + 시그너처 형식 검증
- 작업 주체: Claude Opus 4.7
- 작업 내용:
  - 대시보드 보안 상태 카드의 4번째 항목(AES/CMAC/Ed25519)이 항상 "상태 확인 불가" 하드코딩 → BMU 레코드의 `signature` 필드 존재 여부로 분기
  - 5Hz 송신 주기를 고려해 "최근 60초 이내 서명 수신" 윈도우로 판정 (1초 단위 임계는 RTOS jitter로 흔들림)
  - bmu-agent `POST /api/bmu/data`에 signature 형식 검증 추가 (length=128, lowercase hex), 임의 placeholder/cURL 우회 차단
  - 근거: BMU 펌웨어가 [CMAC verify → CBC decrypt → FC window → Ed25519 sign] 4단계 직렬 실행, 하나라도 실패 시 [SIGN] 미출력 → signature 존재 = 4계층 묶음 통과 (임베디드 세션 합의)
- 변경 파일: `webapp/frontend-react/src/pages/DashboardPage.tsx`, `bmu-agent/routes/bmu.routes.js`
- 커밋: `67b1aad`, `0f49352`.
