# Cycle 04 — Domain-Specific Information Architecture

Based on: Spherity (public passport view), DPP Hero (authoring workflow), 
Bosch (static+live data), Circularise/Siemens (traceability), Trace4EU (compliance).

## Key IA changes:
1. Passport detail: GBA-21 compliance score as hero metric (not buried in tab)
2. Passport detail: live data (SOC/SOH) separate from static specs  
3. Passports list: GBA compliance % visible per row
4. Dashboard: compliance overview across fleet
5. Materials: traceability chain visualization

## Hypothesis
Making compliance data front-and-center (like Spherity/Battery Pass) will distinguish this
from a generic admin panel and make the domain identity unmistakable.
