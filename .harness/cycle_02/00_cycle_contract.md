# Cycle 02 Contract — Dashboard Layout Differentiation

## Objective
Make the dashboard look structurally different from the passports registry and detail pages. Currently all pages use the same Double-Bezel card grid pattern.

## Why This Matters
The master plan identified "uniform rhythm" as the #1 AI-slop pattern. Every page opens identically with eyebrow → heading → cards → table. The dashboard must break this pattern.

## Cycle Hypothesis
A dashboard that uses a compact summary strip + inline data visualization + dense table will look fundamentally different from the card-grid registry and the certificate-style detail page.

## Expected Visible Improvement
- Dashboard has NO Double-Bezel cards for KPIs (use inline metrics)
- Dashboard status visualization is structurally different from detail page gauges
- Dashboard recent passports is a compact row format, not a full-width table
- Navigation header on dashboard shows a distinct "control room" feel

## Risks
- Making dashboard too minimal (losing useful data)
- Breaking data bindings while restructuring layout

## Direction: PIVOT from uniform card grid
